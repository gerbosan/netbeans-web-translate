Configurando el IDE NetBeans para C/C++/Fortran

Febrero 2013 [Revisión número: V7.3-1]

Este documento proporciona información sobre la descarga, instalación y configuración de soporte de C/C++ en el IDE NetBeans. Antes de comenzar a desarrollar en C/C++, el IDE NetBeans require el módulo plugin C/C++ y compiladores C/C++ de terceros, utilidades make y debuggers.

Contenido

  • Habilitando C/C++/Fortran en el IDE
  • Instalando y Configurando los Compiladores y Herramientas
  • Verificando la Instalación
  • Solucionando Problemas en la Herramienta

Para continuar con este tutorial, necesita el siguiente software y recursos.

Software o Recurso Version Requerida
IDE NetBeans 7.3 con el plugin C/C++ de NetBeans
Java Developer Kit (JDK) 6 o superior

Habilitando C/C++/Fortran en el IDE

El IDE NetBeans es un IDE modular dinámico, puede ser modificado añadiendo y quitando módulos del programa. Puede añadir funcionalidades añadiendo módulos plugin o quitar funcionalidades desintalando módulos plugin. Revise la ayuda en línea del IDE NetBeans para más información acerca de los plugins.

Si desea trabajar con programas en C, C++, o Fortran con el IDE NetBeans, su instalación de NetBeans debe incluir el plugin C/C++.

Aunque el plugin se llame C/C++, también incluye soporte para programación en Fortran.

Si todavía no ha instalado el IDE NetBeans 7.3

Si todavía no ha instalado el IDE NetBeans, descargue el paquete con C/C++ o el paquete con Todo de la Página de Descarga del IDE NetBeans 7.3. Si no va a usar otros lenguajes como Java y Ruby, debe descargar el paquete C/C++.

Si ya tiene el IDE NetBeans 7.3 pero desconoce si tiene el plugin C/C++

Si ya tiene el IDE NetBeans, verifique que el IDE NetBeans incluye el plugin C/C++ seleccionado Archivo > Proyecto Nuevo. Si C/C++ está listado como una de las Categorías, ya dispone del módulo plugin C/C++. Debe pasar a la sección Instalando y Configurando los Compiladores y Herramientas.

Si ya tiene el IDE NetBeans 7.3 sin el plugin C/C++

Si su IDE NetBeans no muestra una categoría de proyectos C/C++ cuando selecciona Archivo > Proyecto Nuevo, complete los siguientes pasos para añadir el módulo plugin C/C++.

  1. Si su red usa un proxy, seleccionedel menú del IDE Herramientas > Opciones > General, marque Configuración Manual de Proxy, ingrese los datos de su proxy en Proxy HTTP y Puerto y haga clic en Aceptar.
  2. Seleccione Herramientas > Plugins.
  3. En la caja de dialogo Plugins, haga clic en la lengüeta Plugins Disponibles, y desplace el cursor hasta la categoría C/C++.
  4. Seleccionar el checkbox C/C++ y haga clic en Instalar para iniciar el Instalador del IDE NetBeans.
  5. Haga clic en Siguiente en el Instalador del IDE NetBeans.
  6. Leer el Acuerdo de Licencia, seleccionar el checkbox para aceptar los términos del acuerdo de licencia y haga clic en Siguiente.
  7. Haga clic en Instalar.
  8. Después de que termine la instalación, selecionar Reiniciar el IDE ahora o Reiniciar el IDE luego y haga clic en Terminar.

Instalando y configurando el Compilador y Herramientas

El módulo C/C++ de NetBeans requiere un compilador C compiler, compilador C++, utilidad make, y debugger gdb. Siga las siguientes instrucciones para la plataforma de su sistema de desarrollo.

-  Microsoft Windows
-  Oracle Solaris
-  Linux
-  Macintosh OS X

Microsoft Windows

El módulo C/C++ de NetBeans ha sido probado con compiladores de Cygwin y MinGW. Si instala ambos, Cygwin y MinGW, se cuidadoso al mantener separada la localizacion de las instalciones y de no mezclar las herramientas de Cygwin y MinGW en una colección de herramientas en el IDE.

Nota. Si quiere usar Qt con Windows, debe usar MinGW. Vea el artículo Working with Qt Applications por las instrucciones para instalar MinGW y software de Qt.

-  Compiladores y Herramientas de Cygwin
-  Compiladores y Herramientas de MinGW

Compiladores y Herramientas de Cygwin

El módulo C/C++ de NetBeans ha sido probado con los siguientes compiladores y herramientas de Cygwin.com.

Software o Recurso Version Probada Descripción
cygwin1.dll 1.7.7, 1.7.9 Ambiente similar a Linux de Cygwin para Windows
gcc 4.3.4 Compilador C de Cygwin
g++ 4.3.4 Compilador C++ de Cygwin
gdb 6.8 Debugger GNU de Cygwin
make 3.81 Utilidad make de Cygwin

Si ya cuenta con los compiladores gcc y g++ de Cygwin, make de GNU, y el debugger gdb instalado en su sistema Windows y su ruta está configurada correctamente, asergurese de contar con las versiones correctas.

Para revisar las versiones de compiladores y herramientas de Cygwin:

  1. Escriba los siguientes comandos en el símbolo de sistema de Windows para revisar la versión del ambiente de Cygwin:

    C:\> cygcheck -c cygwin
  2. Escriba los siguientes comandos en el símbolo de sistema de Windows para revisar la versiónde compiladores gcc y g++, make y gdb de Cygwin: 
    C:\> gcc --version
C:\> g++ --version
C:\> make --version
C:\> gdb --version

Si cuenta con las versiones correctas, no es requerida la configuración. Vea Verificando la Instalación para verificar que las herramientas están instaladas correctamente para el IDE de NetBeans.

Para instalar los compiladores gcc y g++, make, el debugger gdb de GNUT de |cygwin.com:

  1. Revise Cygwin User's Guide para información complera acerca de la instalaciín y uso de Cygwin.
  2. Descargue el programa setup.exe de Cygwin haciendo clic en Install Cygwin en la barra de navegación izquierda o haciendo clic directamente en el enlace setup.exe.
  3. Ejecute el programa setup.exe. Acepte las opciones por defecto hasta que llegue a la página Select Your Internet Connection. Seleccionar la mejor opción para su sistema. Haga clic en Next.
  4. En la página Choose Download Site, escoger el sitio de descarga que crea que está más próxima. Haga clic en Next.
  5. En la página Select Packages, selecciona los paquetes a descargar. Haga clic en + a lado de Devel para expandir la categoría de herramientas de desarrollo. Puede ser necesario agrandar la ventana para ver más información.
  6. Seleccione cada paquete que desee descargar haciendo clic en la etiqueta Skip, que revela el número de versión de los paquetes a descargar. Como mínimo, seleccione
    • gcc-core: C compiler
    • gcc-g++: C++ compiler
    • gdb: The GNU Debugger
    • make: the GNU version of the 'make' utility
    Los paquetes que son requeridos por los paquetes que elija serán seleccionados automáticamente.
  7. Haga clic en Next para conectarse al sitio de descarga y descargar los paquete elegidos y haga clicen Finish cuando la instalación esté completa.
  8. Añada el directorio del compilador Cygwin a la ruta de su sistema para permitir que el IDE NetBeans encuentre la colección de herramientas:

    1. Abra el Panel de Control:
      - En Windows XP elija Inicio > Configuración > Panel de Control y haga doble clic en Sistema.
      - En Windows 7, escriba var en la caja de búsqueda del menú Inicio para encontrar rápido el enlaces para Editar las variables de entorno del sistema.
    2. Elija la lengüeta Avanzado y haga clic en Variables de Entorno.
    3. En el panel de Variables de Sistema del dialogo Variables de Entorno, elija la variable Path y haga clic en Editar.
    4. Añada la ruta del directorio-cygwin\bin en la variable Path y haga clic en Aceptar. Por defecto el, directorio-cygwin es C:\cygwin. Los nombres de los directorios deben estar separados con punto y coma. La ruta editada debe verse como %SystemRoot%\system32;%SystemRoot%;C:\Program Files\QuickTime\QTSystem;C:\cygwin\bin
    5. Haga clic en Aceptar en el dialogo de Variables de Entorno y el dialogo de Propiedades de Sistema.
    6. Vea Verificando la Instalación para verificar que las herramientas están instaladas correctamente para el IDE NetBeans.

Compiladores y Herramientas MinGW

El IDE NetBeans 7.3 ha sido probado con el Minimalist GNU for Windows (MinGW) y el entorno tipo Unix Minimal System (MSYS). Las versiones probadas y las instrucciones de instalación se muestran a continuación.

Nota. Si desea usar Qt con Windows, vea el artículo Working with Qt Applications para tener las instrucciones para instalar MinGW y software Qt.

Software o Recurso Version Probada Descripción
gcc 4.6.2 Compilador C
g++ 4.6.2 Compilador C++ de MinGW
gdb 7.4 Debugger GNU de MinGW
make 3.81 Utilidad make de MSYS
Notar que el make de MinGW no está soportado

Para instalar compiladores, make y debugger gdb GNU de mingw.org:

  1. Inicie sesión en Windows usando una cuenta con privilegios de administrador de la computadora.
  2. Descargue el instalador MinGW de http://sourceforge.net/projects/mingw.
  3. Ejecute el instalador MinGW, que debe de tener un nombre parecido a mingw-get-inst-20120426.exe.
  4. Acepte el folder destino por defecto C:\MinGW a fin de minimizar cualquier dificultad potencial de usar los compiladores de otra localización.
  5. En el instalador MinGW seleccione los siguientes componentes para instalar:
    • C
    • C++
    • Fortran (si trabajará con programas Fortran)
    • MSYS Basic System
    • MinGW Developer Toolkit
  6. Haga clic en Install y espere unos minutos mientras el programa de instalación descarga los componentes seleccionados.
  7. Haga clic en Finish cuando los paquetes hayan terminado de descargarse.

Debe añadir las rutas a los binarios para las herramientas MinGW y MSYS en su RUTA. Si instaló en las localizaciones por defecto, las rutas son C:\MinGW\bin y C:\MinGW\MSYS\1.0\bin.

Para editar la RUTA de la variable de entorno en Windows:

  1. Abrir la ventaba de Variables de Entorno:
    • En Windows XP y Windows 2000: Clic derecho en Mi Computadora > Propiedades > lengüeta Avanzados > botón Variables de Entorno.
    • En Windows Vista y Windows 7: Clic derechos en Mi Computadora > Propiedades > enlace Configuración Avanzada de Sistema > botón Variables de Entorno.
  2. En la ventana de Variables de Entorno, seleccionar la variable Ruta en la sección Variable de Sistema y haga clic en Editar.
  3. Al final de la ruta, inserte un punto y coma y añada las rutas de los ejecutables para MinGW y MSYS. Use punto y coma entre las rutas y no introduzca espacio alguno. Tenga cuidado de no eliminar el contenido de su RUTA o su computadora no podrá funcionar correctamente.

    Cuando termine, su ruta se verá parecido a:

    %SystemRoot%\system32;%SystemRoot%;%SystemRoot%\System32\Wbem;
%SYSTEMROOT%\System32\WindowsPowerShell\v1.0;C:\MinGW\bin;C:\MinGW\MSYS\1.0\bin
  4. Haga clic en Aceptar en la ventana de Variables de Entorno.

Vea Verificando la Instalación para verificar que las herramientas se instalaron correctamente para el IDE NetBeans.

Oracle Solaris

En las plataformas Oracle Solaris, puede usar herramientas GNU o herramientas Oracle Solaris Studio. Las herramientas GNU están incluidas en Oracle Solaris 10 y Oracle Solaris 11 por defecto en el directorio /usr/sfw/bin. El software de Oracle Solaris Studio es una suite de herramientas para desarrolladores que es gratuita para descargar. Los compiladores de Oracle Solaris Studio están optimizados para hardware de Oracle Sun y la producción de binarios de alto rendimiento para Oracle Solaris.

El IDE NetBeans 7.3 ha sido probado con los siguientes compiladores y herramientas.

Software o Recurso Versión Probada Descripción

CC 5.10, 5.11, 5.12 Compiladores C++ de Sun Studio 12 Update 1, Oracle Solaris Studio 12.2 y Oracle Solaris Studio 12.3
gcc 3.4.3, 3.4.6 Compiladores GNU C
g++ 3.4.3, 3.4.6 Compiladores GNU C++
gdb 6.8 Debugger GNU

Compiladores de Oracle Solaris Studio 12.3

Si desea usar los compiladores de Oracle Solaris Studio 12.3 en Oracle Solaris 10 o Oracle Solaris 11:

  • Si tiene instalado el software Oracle Solaris Studio 12.3, debe asegurarse que /directoriodeinstalación/solarisstudio12.3/bin está en su ruta antes de iniciar el IDE NetBeans. La localización por defecto en Solaris 10 y Solaris 11 es /opt/solarisstudio12.3/bin
  • Si no tiene instalado el software Oracle Solaris Studio 12.3, puede descargarlo de manera gratuita en http://www.oracle.com/technetwork/server-storage/solarisstudio/downloads/index-jsp-141149.html.

Para descargar e instalar los compiladores de Oracle Solaris Studio 12.3 en Oracle Solaris 10:

  1. Crear un directorio para el archivo descargado. debe tener privilegios de esritura en el directorio.
  2. Descargar el archivo instalador de paquetes para su plataforma en el directorio de descarga.
  3. Ir al directorio de descarga y descomprimir y desempaquetar el archivo descargado. 
    bzcat filename | /bin/tar xvf -
  4. Siga las instrucciones en el Capítulo 2 de Oracle Solaris Studio 12.3 Installation Guide para instalar el compilador C, C++ compiler y parches requeridos de Solaris.
  5. Edite el PATH para añadir la ruta al software de Oracle Solaris Studio 12.3 antes de utilizar el IDE NetBeans.
  6. Vea Verificando la Instalación para verificar qie la herramienta ha sido instalada correctamente para el IDE NetBeans.

Para instalar los compiladores de Oracle Solaris Studio 12.3 en Oracle Solaris 11:

  • Utilice la dirección Oracle Solaris Studio download page y siga las instrucciones de la columna de Instalación de Paquete de Oracle Solaris 11.

Los Compiladores GNU y GNU make

Si quiere hacer uso de los compiladores GNU y GNU make:

  • Si tiene una instalación estándar de Oracle Solaris 10 o Oracle Solaris 11, los compiladores y gmake están instalados en /usr/sfw/bin. Asegúrese que esta localización esta en su PATH antes de iniciar el IDE NetBeans.
  • Si los compiladores y gmake no están instalados en su sistema, los puede descargar desde http://www.sunfreeware.com.

Para descargar e instalar los compiladores GNU compilers y la utilidad make

  1. Descargar gcc y make.
  2. Si los archivos zip no son extraídos automáticamente durante la descarga, los descomprime con gunzip.
  3. Instalar los paquetes con el comando pkgadd.
  4. Debe asegurarse de incluir la carpeta del compilador GNU y la carpeta de GNU make en la ruta de su sistema antes de iniciar el IDE de NetBeans.
  5. Vea Verificando la Instalación para verificar que las herramientas se instalaron correctamente para el IDE NetBeans.

Debugger gdb

Sin importar si usa los compiladores de Oracle Solaris Studio y make de Solaris o los compiladores de GNU y makede GNU, necesita tener el debugger gdb para depurar aplicaciones en el IDE NetBeans. Puede descargar gdb de http://www.sunfreeware.com.

Para descargar e instalar gdb para Oracle Solaris 10:

  1. Descargar gdb 6.8.
  2. Si el archivo zip descargado no es automáticamente extraído durante la descarga, lo puede descomprimir con gunzip.
  3. Instalar el paquete usando el comando pkgadd.
  4. Se debe asegurar de incluir la ruta de gdb en el PATH de su sistema antes de iniciar el IDE NetBeans.
  5. Vea Verificando la Instalación para verificar que las herramientas se instalaron correctamente para el IDE NetBeans.

En Oracle Solaris 11, puede instalar gdb 6.8 desde el repositorio pkg.oracle.com.

Compiladores Oracle Solaris Studio 12.2 en Linux

Si desea usar los compiladores de Oracle Solaris Studio 12.3 en el SO Linux:

  • Si el software Oracle Solaris Studio 12.3 está instalado, se debe de asegurar que/carpeta de instalación/solarisstudio12.3/bin está en la ruta de su sistema antes de iniciar el IDE NetBeans. La localización por defecto para los paquetes en Linux es /opt/oracle/solarisstudio12.3/bin.
  • Si no ha instalado el software Oracle Solaris Studio 12.3, puede lo puede descargar gratuitamente desde http://www.oracle.com/technetwork/server-storage/solarisstudio/downloads/index-jsp-141149.html.

Para descargar e instalar los compiladores de Oracle Solaris Studio 12.3:

  1. Crear una carpeta para el archivo descargado. Debe de tener privilegios de escritura para esa carpeta.
  2. Si su navegador web está configurado para descargar a una ubicación particular como su Escritorio o la carpeta Descargas, configure las preferencias de su navegador web para descargar a la carpeta que ha creado. Para Firefox, la carpeta de descarga está en Editar > Preferencias > Principal.
  3. Descargar el archivo para su plataforma en la carpeta creada. Si está usando Ubuntu, debe descargar el archivo empaquetado en la carpeta donde lo quiera instalar por que los paquetes son para Oracle Linux, Red Hat Enterprise Linux y SuSE Linux Enterprise System. Considerar también que Ubuntu no es una plataforma oficialmente soportada por el IDE de Oracle Solaris Studio pero los compiladores han sido probados para su uso en el IDE NetBeans.
  4. Vaya a la carpeta de descarga, descomprima y desempaquete el archivo descargado. 
    bzcat filename | tar xvf -
  5. Si ha descargado los paquetes RPM, siga las instrucciones en el Capítulo 2 de Oracle Solaris Studio 12.3 Installation Guide para instalar el compilador C, el compilador C++ y el compilador Fortran.
  6. Edite el PATH para añadir la ruta al software de Oracle Solaris Studio antes de iniciar el IDE NetBeans.
  7. Vea Verificando la Instalación para verificar que las herramientas se han instalado correctamente para el IDE NetBeans.

Para desargar e instalar el depurador GNU en Oracle Linux o Red Hat Enterprise Linux, escriba:

yum install gdb

Para descargar e instalar el depurador GNU en Ubuntu, escriba:

sudo apt-get install gdb

Verifying the Installation

To verify that the installation is correct, start the NetBeans IDE, build a sample project, and run it in the gdb debugger.

Start the NetBeans IDE

To start the IDE, do one of the following:

  • Double-click the NetBeans IDE icon on the desktop.
  • On Linux or Solaris platforms, navigate to the bin subdirectory of your NetBeans installation and type ./netbeans.
  • Launch the NetBeans IDE through the desktop menu.
    This method is available on Windows platforms, but might not be available on the Oracle Solaris 10 and Oracle Solaris 11 platforms and some Linux platforms.

Build a Sample Project to Test the Tools

On all platforms, you can build a sample project to test the compiler installation, and run it in the debugger to test the gdb installation.

To build a sample project and run it in the debugger:

  1. Open the New Project wizard by choosing File > New Project.
  2. In the Categories panel on the Choose Project page of the wizard, expand the Samples category and select the C/C++ subcategory.
  3. In the Projects panel, select the Welcome project. Click Next.
  4. On the Project Name and Location page, click Finish.
  5. In the Projects window, right-click the Welcome_1 project and choose Build. If your compilers and make utility are installed correctly and the path to them is set, build output is displayed in the Output window and the project builds successfully.
  6. Expand the Source Files node of the project and double-click the welcome.cc file to open it in the Source Editor.
  7. Set a breakpoint by clicking in the left margin of the Source Editor window next to any line.
  8. Right-click the project and choose Debug. If the gdb debugger is installed correctly and the path to it is set, gdb starts up, the Debugger tabs are displayed, and the Welcome application runs and stops at the breakpoint.
  9. Choose Debug > Continue to run the application to completion.
  10. If the project doesn't build or debugger doesn't work, see the next section.

Troubleshooting Tool Issues

  1. Select Tools > Options and click C/C++ in the top panel of the Options dialog box.
  2. In the Build Tools tab, look at the Tool Collection list to see if your tool collection is listed.
  3. Select the tool collection if it is listed, and check the paths to the tools. If the Base Directory path is incorrect, you may have made a mistake when setting your path environment variable. Refer back to the instructions for setting the path in the section for your platform in this document, and correct the path if necessary.
  4. If the tool collection is not listed, click Restore Defaults. This will cause the IDE to rescan your environment to look for tools and the tool collection should be found if the path environment variable is correct.
  5. If you are still having an issue, try adding a new tool collection and specifying the path to the tools, as follows:
    1. Click Add below the Tool Collection list.
    2. Browse to the directory that contains the executables for the compilers, debugger, and make utility. This is usually the bin directory.
    3. Select the directory and click Open. The IDE should be able to determine the family of the tools and update the other fields in the dialog box appropriately. If not, an error is displayed.
    4. If the tools are identified correctly, click OK in the Add Tool Collection dialog box. The new collection should be displayed in the list.
    5. Select the new tool collection and click Default to make sure this tool collection is used automatically for new projects.

    If you cannot solve the problem, ask for help at the NetBeans C/C++ User Forum.
